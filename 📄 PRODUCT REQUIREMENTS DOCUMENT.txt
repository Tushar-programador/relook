📄 PRODUCT REQUIREMENTS DOCUMENT (PRD)
🏷️ Product Name

FeedSpace (working name)

📌 Category

UGC Collection + Feedback + Testimonial SaaS

🎯 Objective

Enable businesses (especially social-first brands) to:

Collect customer feedback (video, audio, text)
Manage and moderate it
Showcase it via embeddable widgets
Convert feedback into reusable marketing content
🧠 PROBLEM STATEMENT

Small businesses (especially Instagram-first brands):

Receive feedback in DMs (unstructured)
Cannot reuse testimonials easily
Lack tools for collecting video/audio feedback
Have no central dashboard
💡 SOLUTION

A platform that provides:

A public feedback page
A dashboard for management
A widget/iframe for display
A media-first experience (video/audio)
🎯 TARGET USERS
Primary
Instagram brands
D2C startups
Freelancers
Secondary
SaaS companies
Agencies
🧩 CORE FEATURES
1️⃣ AUTH & USER MANAGEMENT
Description

Users can register/login and manage multiple projects.

Requirements
Email/password login
JWT authentication
Multi-project support
APIs
POST /auth/register
POST /auth/login
GET /auth/me
2️⃣ PROJECT / BRAND MANAGEMENT
Description

Each user can create multiple brands/projects.

Features
Create/edit/delete project
Unique slug (used in public URL)
Branding customization
Functional Requirements
Slug must be unique
User can only access own projects
Data Model
{
  "_id": "ObjectId",
  "userId": "ObjectId",
  "name": "string",
  "slug": "string",
  "logo": "string",
  "theme": {
    "primaryColor": "string",
    "accentColor": "string"
  },
  "createdAt": "Date"
}
3️⃣ PUBLIC FEEDBACK COLLECTION PAGE
URL
/feedback/:slug
Description

Public page where customers submit feedback.

Features
No authentication required
Input types:
Video recording
Audio recording
Text input
Optional name/email
Functional Requirements
Must work on mobile (priority)
Max video length (e.g., 60 sec)
Upload progress indicator
4️⃣ MEDIA RECORDING & UPLOAD
Description

Handles video/audio capture and storage.

Requirements
Use MediaRecorder API (frontend)
Upload to cloud storage (R2/S3)
Store metadata in DB
Flow
Client requests upload URL
Upload directly to storage
Save metadata in backend
5️⃣ FEEDBACK MANAGEMENT (DASHBOARD)
Description

Users manage incoming feedback.

Features
View all feedback
Filter by status:
Pending
Approved
Rejected
Approve/reject/delete
Functional Requirements
Only approved feedback appears publicly
6️⃣ EMBED WIDGET SYSTEM
Description

Allow users to display testimonials on their website.

Types
iframe embed
script embed (future)
Features
Show approved feedback only
Layout options:
Carousel
Grid
Example
<iframe src="https://yourapp.com/widget/:slug"></iframe>
